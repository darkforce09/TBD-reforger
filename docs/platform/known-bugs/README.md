# Known bugs

Recorded, triaged defects that are **not currently being actioned** — usually because they sit outside
the realistic usage envelope, are low severity, or are deferred behind other work. The point is a single
place that says exactly what each issue is, why it isn't fixed yet, and what a future session would do to
fix it, so the analysis isn't re-derived from scratch.

Each entry is its own `KB-00x-*.md` file. Add a row here when you file one; flip **Status** to
`Fixed @ <commit>` (or delete the row) when it's resolved.
