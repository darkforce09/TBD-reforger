## In one sentence

Turn height markers from "10 knolls in a JSON nobody can see" into credible, visible peak labels: a zoom band, named peaks/hills merged from `locations.json`, a real prominence floor, and an explicit operator decision on contour labels.

---

## Problem

Audit §6.3. The consumer chain is wired and default-on, but: labels rode the dead text lane (fixed in .12); the data is weak — **10 peaks**, five of them ≤ 55 m knolls (values 29/15/23/21/55), the 375 m entry an injected fallback summit; `dem/peaks.rs` has **no zoom band at all** (`:7-13` — declutter-only, so labels draw at z=−6 island view); the export writes raw `peaks` not the decluttered set (`export-height-labels.mjs:52`); and locations.json's **17 named peaks + 16 named hills** are drawn as *town* labels instead of feeding the heights lane (A8 overlap). Contour index labels were operator-waived (`t152_7_verify_log.md:22`) without a revisit path.

---

## Goal

1. **Zoom band:** `HEIGHT_LABEL_MIN_ZOOM = −2.0`, `HEIGHT_LABEL_MAX_ZOOM = +3.0` in `dem/peaks.rs` (mirror town-band shape; constants locked here, tunable by operator M-row).
2. **Credible peak set:** rebuild `height-labels.json` = named peaks/hills from `locations.json` (name + x/y; elevation sampled from DEM) ∪ DEM prominence peaks with floor `PEAK_MIN_VALUE_M = 80` and true prominence ≥ 15 m; drop the injected-summit hack (real max already qualifies); dedupe against named set by 200 m radius. Labels render `"{Name} · {value} m"` when named, `"{value}"` otherwise.
3. **Export honesty:** exporter writes the post-floor set; sidecar schema gains optional `name`.
4. **Contour decision:** implement contour index labels at `deckZoom ≤ −1` **or** record a fresh operator waiver quote in the verify log — no inherited waiver.
5. Verify log `.ai/artifacts/t152_16_verify_log.md`.
